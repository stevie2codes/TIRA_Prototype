TIRA Reporting — Flow Annotations

Full end-to-end annotation reference for the AI-forward reporting prototype. Each callout is numbered and ready for Figma.

7 Screens 40+ Annotations Prototype Complete March 2026
Homepage → Chat Dialog → Query Card → Refinement → Split View → Report Designer
1

Homepage — Default State ("All Sources")

The landing experience for a returning user. The system knows who they are and personalizes content accordingly.

1.1

Personalized Greeting

"Good morning, Stephen"

The greeting uses the authenticated user's first name and adapts to time of day (morning / afternoon / evening). This replaces the generic "Find the report you need" heading from the logged-out state. The Tyler logo sits alongside the greeting as a brand anchor.

Why: Establishes personal workspace ownership
1.2

Chat Input — Primary Entry Point

The main input field: "Ask a question about your data..."

This is the AI-first entry point. Users describe what they need in natural language rather than navigating a menu tree or building a report from scratch.

Enter key or Send icon → opens chat dialog
1.3

Source Toggle Dropdown

Button in the input toolbar showing the currently selected data source context. Defaults to "All sources."

Clicking opens a dropdown (downward) listing available data sources filtered to the user's role and department permissions. Selecting a source scopes both the input context and the suggestion content below.

The dropdown header shows "Your data sources" with the user's role badge (e.g., "Senior Planner"). Sources the user's department doesn't have access to are not shown. This is permission-driven, not preference-driven.

Click → dropdown opens downward Permission-filtered, not preference-based
1.4

Data Source Dropdown — Open State

Dropdown content when the source toggle is clicked:

  • Header row: "Your data sources" label + role badge ("Senior Planner")
  • "All sources" option: Always first, allows cross-source queries
  • Divider
  • Filtered source list: Each source shows an icon, name, and count of available queries. Only sources matching the user's department appear.

For the prototype persona (Stephen Webb, Senior Planner, Community Development), visible sources are: Permits & Licensing, Code Enforcement, Financial Management, GIS & Mapping.

Role badge reinforces why they see what they see 4 of 7 sources visible for this role
1.5

"Pick up where you left off" — Recent Activity Card

A subtle card surface sitting directly below the input. Contains the user's 3 most recent interactions. Each item shows:

  • Source icon in a small badge (indicates which data domain)
  • Report/query name (primary label, bold)
  • Metadata line (e.g., "Saved report · 2h ago")
  • Arrow on hover — reveals a play icon indicating "click to resume"

Prototype data:

  1. "Building Permits by Month — Downtown Only" — Saved report · 2h ago
  2. "Code Violations Summary" — This morning
  3. "Budget vs. Actuals — FY2025" — Saved report · Yesterday
Why top position: ~60% of sessions start from a recent item Click → reopens in chat dialog
1.6

"Suggested" — Task-Based Action Rows

Three full-width interactive rows below the recent activity card. Each represents a high-value starting question:

  1. "What needs my attention today?" — Flag icon, indigo accent
  2. "Show me what's overdue" — Warning icon
  3. "What changed since yesterday?" — Update icon

Each row has a leading icon badge (indigo background), the question text, and a directional arrow on the right.

Why these aren't chips: These are primary calls-to-action, not passive filters. The larger surface area and full-width layout communicates "click me to start working" rather than "here's a tag you might filter by."

Click → opens chat dialog with this prompt Full-width rows > small chips for primary CTAs
1.7

Browse Row — Category Chips + Library

A compact horizontal row of small pills at the bottom:

  • Category chips: Permits, Violations, Budget, Inspections — each with a leading icon
  • Vertical divider
  • "Saved Reports" chip — folder icon, links to the full Report Library

Clicking a category chip switches the source dropdown to that data source and refreshes the suggestions above. Clicking "Saved Reports" opens the Library dialog.

Compact: navigational shortcut, not primary action Category click → selects source + updates suggestions
1.8

Disclaimer

"AI can make mistakes. Always verify responses."

Standard AI disclaimer. Muted text, small font. Present on every screen involving AI-generated content.

Required for all AI-generated surfaces
2

Homepage — Source-Selected State

When the user selects a specific data source from the dropdown (e.g., "Permits & Licensing"), the content below the input updates.

2.1

Source Toggle — Updated

The toggle button now shows the selected source's icon and name (e.g., "Permits & Licensing" with the description icon) instead of "All sources."

Visual state change reflects active source
2.2

"Recent from [Source]" Card

If the user has recent activity matching the selected source, a filtered recent card appears at the top. Same visual treatment as the default recent card, but scoped.

Example for Permits & Licensing: Shows only "Building Permits by Month — Downtown Only" (the one matching item).

Filtered to selected source only
2.3

"Common queries" — Source-Specific Action Rows

Replaces the task-based suggestions. Shows queries specific to the selected data source, styled as the same full-width action rows.

Example for Permits & Licensing:

  1. Building permits by month
  2. Permit processing times
  3. Pending permit applications
  4. Permits by contractor
  5. Daily inspection queue

Each row uses the source's icon in the leading badge. These are the top 5 things people actually ask for from this dataset — reducing the blank-page problem.

Click → opens chat dialog with that query Pre-built starting points for common workflows
3

Chat Dialog — Conversational Workspace

Full-screen modal dialog that opens when the user submits a query from the input, a suggestion row, or a recent item.

3.1

Chat Header

  • AI icon — circular border with sparkle icon
  • "Report Assistant" title
  • More options icon button (kebab menu)
  • Close icon button (X) — returns to homepage
3.2

User Message

The user's query appears as a right-aligned chat bubble using <forge-ai-user-message>.

Example: "Show me building permits issued by month, broken down by district"

3.3

Thinking Indicator

After a 500ms delay, a <forge-ai-thinking-indicator> appears. Displayed for ~2 seconds before the response appears.

The 500ms pause prevents flicker on fast responses. The 2-second processing time is the prototype simulation.

500ms delay → 2s visible → response
3.4

Query Card — AI Response

The centerpiece of the chat experience. Replaces a flat text response with a structured, progressive-disclosure card.

3.4a Card Header

AI sparkle icon + Report title (bold) + Metadata row: Data source with database icon · Freshness with green dot (e.g., "Updated 2 hours ago") · Row/column count.

Why metadata is inline: Users in government reporting need to know immediately where data came from and how fresh it is. This is a trust requirement — stale or wrong-source data has real compliance consequences.

3.4b AI Summary

Markdown-formatted narrative summary. Highlights key findings with bold text. Example: "I found 2,847 building permits issued across 5 districts over the past 12 months."

Gives executives the headline answer. Technical users scan it, then dive into the data.

3.4c Mini Data Preview

Compact table: first 3 rows × 4 columns. Below: "9 more rows · 3 more columns · Explore full results."

Lets users visually confirm "yes, this is the right data" before committing. The truncation creates a pull toward "Explore Results."

3.4d Disclosure Sections (Progressive Disclosure)

Four collapsible sections, all start collapsed:

  1. Data Source — System name, total records, last update
  2. SQL Query — Generated SQL + "Copy SQL" button
  3. Assumptions — Decisions the AI made (e.g., "issue date, not application date")
  4. Data Citations — Specific tables and joins used

Most users don't need SQL. But for analysts, builders, and compliance reviewers, this is critical. Collapsing by default keeps the card scannable.

3.4e Refinement Chips

Suggestion chips below the disclosures: "Filter by date range" · "Break down by permit type" · "Compare to last year" · "Add residential vs. commercial split"

Clicking triggers the three-tier refinement system (see Screen 4).

3.4f Action Buttons

  • "Explore Results" (primary, indigo) — Opens the side panel report view
  • "Copy Summary" (secondary, outlined) — Copies AI summary text to clipboard
Every response shows source + freshness Disclosures: click header to expand, chevron rotates 180°
3.5

Chat Footer

Persistent input area: text input ("Ask a question..."), Send button, attachment (+), voice input (mic). User can always type follow-up questions.

4

Refinement — Three-Tier Response System

When a user clicks a refinement chip, the system classifies the request into one of three tiers and responds accordingly.

Tier Triggers Response Previous Card
1. Card "break down," "filter," "group by," "split," "only" New query card with updated data, summary, SQL, and refinement chips Collapses to one-line bar
2. Text "compare," "chart," "trend," "export," "schedule" Narrative text response with data source badge Stays expanded
3. Handoff "cross-reference," "join," "calculated field," "inspector" Handoff card with "Open in Report Designer" action Stays expanded
4.1

Tier 1: "Card" — Data-Shaping Refinement

Chip clicked → Previous cards collapse → User message → Thinking (1.5s) → New query card

Collapsed card state: Previous card reduces to a single bar — AI sparkle icon, report title, condensed metadata. Click to re-expand.

Example: "Break down by permit type" → Previous card collapses → New card: "Building Permits by Month — By Permit Type" with Residential Reno, Commercial New columns.

Why collapse: prevents scrolling past walls of old content Click collapsed bar → re-expands to full card
4.2

Tier 2: "Text" — Narrative Response

Previous card does NOT collapse. User message appears, followed by a text-only response with markdown formatting and a small data source badge.

Example: "Compare to last year" → "Total permits are up 14%, driven primarily by a 22% increase in residential activity..."

No new card: adds context to existing data, doesn't replace it
4.3

Tier 3: "Handoff" — Designer Escalation

AI responds with an explanation of why this exceeds chat capabilities, then shows a handoff card:

  • Header: "This needs the Report Designer"
  • Context: Dataset name, "Filters and query will transfer," "SQL query ready to edit"
  • Actions: "Open in Report Designer" (primary) + "Request from Tyler" (secondary)
Transparent about limits, clean escalation path All context transfers — user doesn't start over
5

Split View — Report Panel

When the user clicks "Explore Results," the dialog transitions to a two-panel split view.

5.1

Layout Transition

Full-width chat view animates into a two-panel split:

  • Left panel (450px, resizable 320–600px): Chat conversation preserved
  • Divider: Draggable vertical handle
  • Right panel (flex remaining): Report canvas

The "Explore Results" button is replaced with a "Report open in side panel" status badge.

Divider draggable with min/max constraints
5.2

Left Panel: Chat (Condensed)

Chat continues in left panel: same header, all messages preserved (scrollable), input footer at bottom. User can keep chatting while viewing the report.

5.3

Report Panel: Title Bar

  • Report title (bold)
  • Data source badge with freshness
  • Open in new window icon button
  • Close icon button — collapses back to full-width chat
5.4

Report Panel: Action Bar

Left — View Toggle: Table (default) · Chart · SQL

Right — Report Actions dropdown:

  • Export — PDF, CSV, or Excel
  • Schedule — Recurring delivery
  • Share — Send to colleagues
  • Save to Library
  • Divider
  • Open in Report Designer — Advanced layout, joins, formatting
View toggle switches between table/chart/SQL Report Actions is a dropdown with click-to-dismiss
5.5

Report Panel: Filter Bar

"Filters" label + chips: "All Districts," "All Months," "Permit Type" — each with dropdown arrows. Quick inline filtering without returning to chat.

5.6

Report Panel: Table View (Default)

Forge <forge-table> with dense mode, fixed headers, all columns sortable, all data rows. This is the full dataset, not the 3×4 mini preview.

5.7

Report Panel: Chart View

Horizontal bar chart. Each data row becomes a labeled bar, width proportional to max value.

Prototype: simplified bar chart. Production: chart type selection.
5.8

Report Panel: SQL View

Generated SQL in a <pre><code> block. For developers and power users to inspect, copy, or reference the exact query.

6

Report Designer (Handoff)

Launched from the handoff card or Report Actions dropdown.

6.1

Context Transfer

Before mounting the designer, the system writes full context to sessionStorage: report title, data source, freshness, SQL, columns, data, user query, AI summary, transparency metadata, and handoff reason.

sessionStorage: clean same-tab transfer to React app
6.2

Loading State

"Back to Chat" button (functional immediately), "Loading Report Designer..." label, thinking indicator.

Back button works before load completes — user can bail
6.3

Designer Interface

React Flow + dnd-kit based designer mounts inside dialog. Navigation bar with "Back to Chat" button. Full drag-and-drop canvas.

"Back to Chat" unmounts React and restores the chat conversation exactly as it was. No data is lost.

6.4

Error State

If React fails to load: error icon, "Failed to load Report Designer" message, "Reload" button.

7

Saved Reports Library

Opened from the "Saved Reports" chip on the browse row or "Open in Chat" on library cards.

7.1

Library Header

Close button (X), "Saved Reports Library" title, search input with icon.

7.2

Category Filters

Horizontal chip row: "All" (default active) + category chips derived from reports.

7.3

Library Stats

Summary line: "[count] reports · [count] scheduled · [count] standard, [count] custom"

7.4

Report Cards Grid

Each saved report card shows: type badge + category, title, description, metadata (schedule, owner, row count, last run), and three action buttons:

  • "Run Now" — Executes with latest data
  • "Open in Chat" — Loads into conversational workspace
  • "Open in Designer" — Opens in Report Designer

Cross-Cutting Patterns

Consistent interaction patterns used across multiple screens.

A Data Source Badge

Database icon + source name + green dot + freshness. Appears on query cards, text responses, report panel. Every AI-generated content must show its source.

B Thinking Indicator

<forge-ai-thinking-indicator> with show-text. Staged timing: 500ms delay before showing, 1.5–2s visible.

C Copy-to-Clipboard

Used for SQL and summaries. Icon → checkmark, label → "Copied!", reverts after 2 seconds.

D Progressive Disclosure

Query card disclosures + report actions dropdown. Default collapsed. Chevron rotates on expand. Smooth scroll to content.

E Role-Based Filtering

Content filtered by department at the service layer before rendering. Affects sources, suggestions, and recents.

F sessionStorage Handoff

Bridges vanilla JS chat → React Report Designer. Structured JSON payload written before mount, read and cleared on init.
